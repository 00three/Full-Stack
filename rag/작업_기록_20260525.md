## 2. 코드 변경 사항

### A. PDF 텍스트 제거 (데이터 품질)

**문제**: 전체 chunk의 62% (5414/8805)가 깨진 PDF 추출 텍스트. 검색 결과 오염.

**변경 파일:**

| 파일 | 변경 |
|---|---|
| `Crawler/src/crawlers/kcc_crawler.py` | `process_attachments()` 호출 + `attachment_text=` 인자 제거 |
| `Crawler/src/crawlers/nsp_crawler.py` | 동일 |
| `Full-Stack/rag/preprocess.py` | `split_inputs()`에서 `pdf_text` 분기 제거, docstring 업데이트 |
| `Full-Stack/rag/schema.sql` | `data_type CHECK`에서 `'pdf_text'` 제거 |
| dev DB | `DELETE FROM documents WHERE data_type='pdf_text'` (5414건 삭제) |

**결과**: 3391개 body_text chunk만 남음. 깨진 PDF 노이즈 완전 제거.

### B. 문장 기반 청킹 (chunker.py 전면 교체)

**문제**: `RecursiveCharacterTextSplitter(chunk_size=400)` → 400자 하드컷으로 문장 중간 절단.

**변경 파일:**

| 파일 | 변경 |
|---|---|
| `Full-Stack/rag/chunker.py` | LangChain 기반 → 한국어 문장 기반 청킹으로 전면 교체 |

**새 방식:**
- 한국어 종결어미(다/요/죠/음/됨/함/까) + 마침표 기준 문장 분리
- `target_size=600` (소프트 리밋, 문장 경계에서 끊음)
- `max_size=1200` (하드 리밋, 예외적 강제 분할)
- `overlap_sentences=1` (글자 겹침 대신 문장 1개 겹침)
- 환경변수로 조정 가능: `CHUNK_TARGET_SIZE`, `CHUNK_MAX_SIZE`, `CHUNK_OVERLAP_SENTENCES`
- `langchain_text_splitters` 의존성 제거됨

### C. 참고기사 검색 fallback 추가 (rag_service.py)

**문제**: NSP만 `parent_doc_id` 연결이 있어서, KCC/MBC/NODONG 보도자료 선택 시 참고기사 0건.

**변경 파일:**

| 파일 | 변경 |
|---|---|
| `Full-Stack/backend/adapters/rag_service.py` | `_linked_references()` 빈 결과 시 `_fast_keyword_related()`로 fallback |

**이전:**
```python
if source_release_id:
    results = self._linked_references(source_release_id)
# linked 없으면 → 빈 결과 그대로 반환
```

**변경 후:**
```python
if source_release_id:
    results = self._linked_references(source_release_id)
if not results and query_text:
    results = self._fast_keyword_related(...)  # fallback
```

### D. 키워드 검색 정확도 개선 (rag_service.py)

**문제**: `_fast_keyword_related()`가 단일 단어 OR 매칭 → 무관한 결과 다수.

**변경 내용:**
- 구문 단위 추출 (2어절 구문 우선)
- 단일 단어는 3글자 이상만
- 불용어 목록 확대 (22개 → 32개)
- **AND 조건 강화**: 최소 2개 키워드 매칭 필수 (`_RELATED_MIN_MATCH=2`)
- 점수 가중치: title 매칭 6점, body 매칭 2점

---

### E. tsvector 키워드 검색 개선 (search.py)

**문제**: `plainto_tsquery`가 모든 단어를 AND로 묶어 제목 길이 쿼리에서 매칭 0건. `'simple'` 컨피그로 한국어 조사 분리 못 함.

**변경 파일:**

| 파일 | 변경 |
|---|---|
| `Full-Stack/rag/search.py` | `keyword_search()` 전면 개선 |

**변경 내용:**
- kiwi 한국어 형태소 분석기로 쿼리에서 명사 추출
- "캘리포니아 주지사가 AI 일자리..." → `['캘리포니아', '주지사', 'AI', '일자리', ...]`
- `plainto_tsquery` (AND) → `to_tsquery` (OR)
- `ts_rank`로 매칭 많을수록 높은 점수

**결과 (매칭 chunk 수):**

| 쿼리 | 이전 | 개선 후 |
|---|---|---|
| T04 홈쇼핑 | 1건 | 271건 |
| T06 AI | 1건 | 317건 |
| T02 가자 | 0건 | 121건 |
| T08 TBS | 0건 | 222건 |
