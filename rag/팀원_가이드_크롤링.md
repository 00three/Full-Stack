# 크롤링 담당 가이드

> 현재 구현 기준의 RAG 연동 입력 명세

## JSONL 1줄 형식

```json
{
  "doc_id": "kcc_20260421_0eaf3e36",
  "source": "KCC",
  "document_kind": "press_release",
  "parent_doc_id": null,
  "department": "사무처",
  "author": "임현주",
  "title": "이동통신 3사 위치정보, 정확도 개선됐다",
  "date": "2026-04-21",
  "summary": null,
  "content_text": "본문 텍스트",
  "attachment_text": "첨부파일 추출 텍스트",
  "detail_url": "https://...",
  "image_urls": [],
  "attachments": [],
  "hashtags": [],
  "references": [],
  "crawled_at": "2026-04-21T12:24:32Z"
}
```

## 필드 규칙

| 필드 | 필수 | 설명 |
|---|---|---|
| `doc_id` | O | 멱등 적재 키. 동일 문서는 항상 같은 값이어야 함 |
| `source` | O | 내부 소스 약어 또는 외부 참고기사 매체명 |
| `document_kind` | O | `press_release` 또는 `reference_article` |
| `parent_doc_id` | X | 외부 참고기사의 원 보도자료 `doc_id` |
| `title` | O | 문서 제목 |
| `date` | X | `YYYY-MM-DD` |
| `content_text` | O | HTML 제거된 본문 |
| `attachment_text` | X | 실제 첨부파일에서 추출한 텍스트 |
| `detail_url` | O | canonical URL |
| `references` | X | 외부 참고 링크 메타데이터 |

## source 정규화

| 크롤러 출력 | ingest 저장값 |
|---|---|
| `KCC` | `KCC` |
| `Broadcast` | `MBC` |
| `National Assembly` | `NSP` |
| `Media Union` | `NODONG` |
| 외부 참고기사 매체명 | 그대로 저장 |

## 배치 전달

- 크롤러는 하루 파일에 append하지 않습니다.
- 회차마다 `batch_*.jsonl` 파일 1개를 outbox에 생성합니다.
- `ingest-worker`가 성공한 batch를 `processed`로 이동합니다.
- 같은 batch를 다시 처리해도 `doc_id` upsert로 멱등성이 유지되어야 합니다.

## 중복 방지 기준

1. URL은 canonicalize 후 저장합니다.
2. `KCC`는 `boardSeq`처럼 안정적인 원본 ID를 우선 사용합니다.
3. 외부 참고기사는 canonical URL 기준으로 dedupe합니다.
4. 저장 직전 batch 내부에서도 `doc_id` 중복을 제거합니다.
