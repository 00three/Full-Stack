# 크롤링 담당 가이드

> RAG 파이프라인 연동을 위한 크롤링 데이터 형식 안내

---

## 크롤링 JSON 형식

```json
{
  "source": "KCC",
  "title": "'청년 크리에이터미디어 산업'에 85억 지원",
  "date": "2026-03-18",
  "detail_url": "https://www.kcc.go.kr/...",
  "content": {
    "body_text": "본문 텍스트...",
    "pdf_text": "PDF에서 추출한 텍스트...",
    "table_markdown": "| 연도 | 이용자 |\n|---|---|\n| 2022 | 1만명 |",
    "table_natural": "2022년 이용자 수는 1만명이다."
  },
  "attachments": [
    {
      "file_name": "보도자료.hwp",
      "file_type": "hwp",
      "download_url": "https://...",
      "extracted_text": "HWP에서 추출한 텍스트..."
    }
  ],
  "content_hash": "sha256해시값"
}
```

---

## 필드 설명

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `source` | string | O | `KCC` / `National_Assembly` / `Broadcast` / `Media_Union` |
| `title` | string | O | 보도자료 제목 |
| `date` | string | O | `YYYY-MM-DD` 형식 통일 |
| `detail_url` | string | O | 원문 링크 |
| `content.body_text` | string | O | 본문 텍스트 (HTML 태그 제거된 순수 텍스트) |
| `content.pdf_text` | string | X | PDF 첨부에서 추출한 텍스트, 없으면 null |
| `content.table_markdown` | string | X | 표를 markdown 형태로 보존, 없으면 null |
| `content.table_natural` | string | X | 표를 자연어로 변환한 텍스트, 없으면 null |
| `attachments` | array | O | 첨부파일 목록, 없으면 빈 배열 `[]` |
| `content_hash` | string | O | body_text의 SHA256 해시 (중복 크롤링 방지) |

---

## DB 테이블 매핑 (raw_documents)

| JSON 필드 | raw_documents 컬럼 | 비고 |
|-----------|-------------------|------|
| (없음) | `id` UUID | DB가 자동 생성 |
| `source` | `source` | 그대로 INSERT |
| `title` | `title` | 그대로 INSERT |
| `date` | `date` | 그대로 INSERT |
| `detail_url` | `detail_url` | 그대로 INSERT |
| `content` | `content` JSONB | 그대로 INSERT |
| `attachments` | `attachments` JSONB | 그대로 INSERT |
| `content_hash` | `content_hash` | 그대로 INSERT |
| (없음) | `created_at` | DB가 자동 생성 |

`id`와 `created_at`은 DB에서 자동 생성되므로 크롤링에서는 위 JSON만 뽑아주면 바로 INSERT 가능.

## source 값 기준

| 소스 | source 값 |
|------|----------|
| 방송통신위원회 | `KCC` |
| 국회 동향 | `National_Assembly` |
| MBC | `Broadcast` |
| 언론노조 | `Media_Union` |

---

## 주의사항

1. **content는 반드시 분리**: body_text / pdf_text / table을 따로 넣어야 나중에 실험 가능
2. **같은 내용 첨부파일**: HWP/PDF/HWPX 동일 내용이면 하나만 텍스트 추출
3. **source 값 통일**: 위 4개 값 중 하나로 고정
4. **date 형식 통일**: 반드시 `YYYY-MM-DD`
5. **content_hash**: 같은 해시면 이미 크롤링된 문서 → 스킵
