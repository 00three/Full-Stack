# 전처리 담당 가이드

> 크롤링 데이터를 받아서 청킹 + 임베딩하여 pgvector에 저장하는 작업 안내

---

## 전체 흐름

```
크롤링 JSON 수신
    ↓
1. 텍스트 합치기: body_text + pdf_text + table_natural
    ↓
2. 텍스트 정제: HTML 태그, 특수문자 제거, 숫자/단위 보존
    ↓
3. Recursive 청킹: 400자 단위, overlap 100자
    ↓
4. Contextual Retriever: 각 chunk에 문서 맥락 prefix 생성 (LLM 호출)
    ↓
5. 메타데이터 prefix 부여: [출처 | 날짜 | 제목]
    ↓
6. full_text 생성: context_prefix + 메타데이터 prefix + original_text
    ↓
7. BGE-M3 임베딩: full_text → 1024차원 dense vector
    ↓
8. 중복 체크: 코사인 유사도 ≥ 0.95 → 메타데이터 병합
    ↓
9. pgvector documents 테이블에 INSERT
```

---

## 1단계: 텍스트 합치기

크롤링 JSON의 `content` 안에 있는 텍스트를 합침:

```python
text = ""
text += content["body_text"] or ""
text += "\n\n" + content["pdf_text"] if content.get("pdf_text") else ""
text += "\n\n" + content["table_natural"] if content.get("table_natural") else ""
```

- `table_markdown`은 합치지 않음 (원본 보존용, 임베딩 대상 아님)
- `table_natural`은 합침 (자연어라 임베딩 가능)

---

## 2단계: 텍스트 정제

- HTML 태그 제거
- 특수문자 정리
- 숫자, 단위, 고유명사는 보존 (예: "85억", "2만 명")

---

## 3단계: Recursive 청킹

LangChain `RecursiveCharacterTextSplitter` 사용:

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=400,
    chunk_overlap=100,
    separators=["\n\n", "\n", ". ", " "]
)

chunks = splitter.split_text(cleaned_text)
```

- 자르는 우선순위: 문단 → 줄바꿈 → 마침표 → 공백
- 400자 안에서 최대한 의미 단위 보존
- chunk_size는 나중에 실험(200/400/600)으로 최적값 도출 예정

---

## 4단계: Contextual Retriever

각 chunk에 대해 LLM을 호출하여 문서 전체 맥락 요약 prefix를 생성:

```
[Before]
"사전동의 예외 허용을 2년 연장한다고 밝혔다"

[After]
"[방통위 2026-03-05 보도자료: 소상공인 점포 정보 전송 서비스 연장]
이 내용은 방통위가 소상공인 지원을 위해 발표한 정책 중 사전동의 예외 조항에 관한 것이다.
사전동의 예외 허용을 2년 연장한다고 밝혔다"
```

- Anthropic(Claude) 논문 기반 기법
- 검색 정확도 대폭 향상 (실패율 49% 감소)

---

## 5단계: 메타데이터 prefix

```
[KCC | 2026-03-18 | 소상공인 점포 정보 전송 서비스]
```

---

## 6단계: full_text 생성

```
full_text = context_prefix + 메타데이터 prefix + original_text
```

이 full_text가 임베딩 대상이자 tsvector 키워드 검색 대상임.

---

## 7단계: BGE-M3 임베딩

```python
from FlagEmbedding import BGEM3FlagModel

model = BGEM3FlagModel("BAAI/bge-m3", use_fp16=True)
embedding = model.encode(full_text)["dense_vecs"]  # 1024차원
```

---

## 8단계: documents 테이블에 INSERT

| 컬럼 | 값 |
|------|-----|
| `id` | UUID 자동 생성 |
| `raw_document_id` | raw_documents 테이블의 id (FK) |
| `chunk_id` | `{짧은 source token}_{date}_{doc hash}_{번호}` (예: `kcc_20260318_ab12cd34_001`) |
| `source` | 원본 문서의 source |
| `date` | 원본 문서의 date |
| `title` | 원본 문서의 title |
| `data_type` | `body_text` / `pdf_text` / `table` |
| `context_prefix` | 4단계에서 생성한 맥락 요약 |
| `original_text` | 원본 chunk 텍스트 |
| `full_text` | 6단계에서 생성한 전체 텍스트 |
| `embedding_dense` | 7단계에서 생성한 1024차원 벡터 |
| `created_at` | 자동 생성 |

---

## chunk_id 형식

외부 참고기사까지 적재하므로 매체명이 길 수 있습니다. `chunk_id`에는 긴 원문 출처 대신 길이 제한을 둔 source token을 사용하고, 문서별 해시를 함께 넣어 64자 제한 안에서 충돌을 피합니다.

```
{source}_{YYYYMMDD}_{3자리 번호}

예시:
KCC_20260318_001
KCC_20260318_002
National_Assembly_20260318_001
```

---

## 주의사항

1. **청킹은 Recursive 방식으로** (문장 단위 분할 X)
2. **임베딩 모델은 BGE-M3** (SentenceTransformer 아님)
3. **정치 분류 불필요**: 전체 문서를 다 임베딩 (필터링 없이)
4. **table_markdown은 임베딩 안 함**: table_natural만 합쳐서 임베딩
5. **chunk_size는 파라미터화**: 나중에 200/400/600 실험 예정
