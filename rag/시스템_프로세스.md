## 2. 데이터 전처리 상세 (ingest 파이프라인)

```
크롤러 JSONL 1줄
│
├── doc_id: "kcc_20260522_3c23ee64"
├── source: "KCC"
├── title: "홈쇼핑 규제 완화로 산업 활력..."
├── content_text: "방송미디어통신위원회는 22일..."  (2363자)
├── attachment_text: null  (PDF 제거 ✅)
│
↓ [1단계: split_inputs]
│
├── body_text: "방송미디어통신위원회는 22일..."
│   (content_text만 사용, PDF 제거됨)
│
↓ [2단계: clean_text]
│
├── HTML 태그 제거
├── 제어문자 제거
├── 공백/개행 정규화
│
↓ [3단계: 문장 기반 청킹 ✅]
│
├── Chunk 1 (499자): "홈쇼핑 규제 완화로 산업 활력..."
├── Chunk 2 (576자): "홈쇼핑은 중소기업의 브랜드..."
├── Chunk 3 (485자): "셋째, 유료방송사와 홈쇼핑사..."
├── Chunk 4 (412자): "넷째, 정액수수료 방송..."
├── Chunk 5 (449자): "기술 환경 변화를 고려해..."
│   (문장 경계에서 분할, 중간 절단 없음)
│
↓ [4단계: kiwi 명사 추출 ✅]  (각 chunk별)
│
├── Chunk 1 keywords: "쇼핑 규제 완화 산업 활력 중소기업 상생 강화 방미통위..."
├── Chunk 2 keywords: "쇼핑 중소기업 브랜드 인지도 성장 기회 판로 송출 수수료..."
├── ...
│
↓ [5단계: BGE-M3 임베딩]  (각 chunk별)
│
├── Chunk 1 embedding: [0.12, -0.45, ..., 0.88]  (1024차원)
├── Chunk 2 embedding: [0.08, -0.32, ..., 0.91]  (1024차원)
├── ...
│
↓ [6단계: 중복 병합]
│
├── cosine ≥ 0.95 매치 있으면 → 기존 chunk에 source_doc_ids 추가
├── 매치 없으면 → 새 chunk INSERT
│
↓ [INSERT into documents]
│
│  chunk_id:        "kcc_20260522_3c23ee64_001"
│  source:          "KCC"
│  title:           "홈쇼핑 규제 완화..."
│  original_text:   "방송미디어통신위원회는 22일..."
│  full_text:       "방송미디어통신위원회는 22일..."
│  keywords_text:   "쇼핑 규제 완화 산업 중소기업..."  ← kiwi 명사
│  embedding_dense: [0.12, -0.45, ..., 0.88]           ← BGE-M3
```

## 3. 검색 프로세스 상세

```
기자가 보도자료 선택 (예: "홈쇼핑 규제 완화")
│
↓ GET /press-releases/related?ids=kcc_20260522_3c23ee64
│
↓ [search_related]
│
├── 1차: _linked_references(doc_id)
│   └── SELECT ... WHERE parent_doc_id = doc_id
│   └── NSP만 결과 있음 (크롤러가 연결해둔 외부 기사)
│
├── 결과 없으면 fallback ✅
│
├── 2차: _fast_keyword_related()
│   ├── 제목에서 키워드 추출: ["홈쇼핑", "규제 완화", "중소기업", "상생"]
│   ├── SQL: title ILIKE '%홈쇼핑%' AND original_text ILIKE '%규제%' ...
│   ├── 최소 2개 키워드 매칭 필수
│   └── 점수: title 매칭 6점 + body 매칭 2점
│
├── (EC2 hybrid_search 사용 시)
│   ├── Dense: BGE-M3 embedding cosine 유사도 → top 50
│   ├── Keyword: keywords_text에서 kiwi OR 검색 → top 50
│   ├── RRF: 두 결과 합산 → top 30
│   ├── 시간 감쇠: 최신 문서 우선 → top 15
│   └── Cross-Encoder rerank → top 10
│
↓ [결과 반환]
│
└── 참고기사 10건 (chunk 단위)
    ├── [1] KCC "2026년 제12차 위원회 결과" (score: 0.035)
    ├── [2] NSP "정부의 규제개혁 대전환" (score: 0.028)
    └── ...
```

## 4. 기사 생성 프로세스 상세

```
기자 선택:
├── 보도자료: "홈쇼핑 규제 완화..." (doc_id)
├── 참고기사: chunk_id 3개 선택
├── 모델: claude-sonnet-4-6
├── 스타일: default / mediaus
├── 톤: default / mz / professional / ...
│
↓ POST /articles/generate/stream
│
↓ [_build_merged_chunks]
│
│  merged[0] = 보도자료 전체 (content_text, 2363자)     ← MAIN
│  merged[1] = 참고기사 chunk A (original_text, ~400자)  ← REF
│  merged[2] = 참고기사 chunk B (original_text, ~400자)  ← REF
│  merged[3] = 참고기사 chunk C (original_text, ~400자)  ← REF
│
↓ [1차 LLM: JSON 추출 — Haiku 4.5] ✅
│
│  시스템: "한국어 보도자료에서 핵심 팩트를 JSON으로 추출하라"
│  입력:   merged 텍스트 (main 1800자 + ref 각 700자까지)
│
│  ┌────────────────────────────────────────────────┐
│  │ {                                              │
│  │   "who": "방송미디어통신위원회(위원장 김종철)", │
│  │   "policy": "홈쇼핑 상생·활력 제고 방안",       │
│  │   "decision": "중소기업 편성비율 단계적 인하",   │
│  │   "target": "홈쇼핑 사업자, 중소기업",          │
│  │   "numbers": "편성비율 55~80%, 데이터 50→25%",  │
│  │   "origin": "모바일 쇼핑 급성장, TV 시청 변화",  │
│  │   "effect": "산업 활력 제고, 상생 기반 강화"     │
│  │ }                                              │
│  └────────────────────────────────────────────────┘
│  
│  실패 시 → fallback (규칙 기반: 본문 잘라넣기)
│
↓ [2차 LLM: 기사 생성 — Sonnet 4.6]
│
│  시스템: 한국어 속보기사 작성 규칙 (제목/리드/본문 구조)
│  입력:   JSON + merged chunks (main 4200자 + ref 각 1200자까지)
│
│  규칙:
│  - 보도자료 중심 60%+ / 참고기사는 배경·맥락
│  - 매 단락 끝에 [1][2] citation 마커 필수
│  - 추측·전망 표현 금지 (단언체)
│  - 출처에 없는 사실 절대 금지
│
│  ┌────────────────────────────────────────────────┐
│  │ 장르: 기관발표                                  │
│  │ 제목: 방미통위, 홈쇼핑 중소기업 편성비율 인하... │
│  │ 리드: 방송미디어통신위원회가 22일...             │
│  │ 본문: (1400~2200자)                             │
│  │   단락1 ... [1]                                │
│  │   단락2 ... [1][2]                             │
│  │   단락3 ... [3]                                │
│  │   ...                                          │
│  └────────────────────────────────────────────────┘
│
↓ [후처리]
│
├── mediaus 스타일이면 바이라인 추가
├── citations 메타데이터 구성
│
↓ [저장]
│
├── generated_articles 테이블 INSERT
├── SSE stream으로 프론트엔드에 실시간 전달
│
↓ [프론트엔드 표시]
│
├── 제목 / 리드 / 본문 렌더링
├── [1][2] 마커 → 녹색 아이콘 → 호버 시 출처 툴팁
└── 저장 (.md 다운로드) / 발행 (비활성화)
```

## 5. 오늘 변경된 부분 표시 (✅)

| 단계 | 변경 내용 |
|---|---|
| 크롤러 | PDF 첨부파일 추출 제거 ✅ |
| 1단계 split_inputs | pdf_text 분기 제거 ✅ |
| 3단계 청킹 | 400자 하드컷 → 문장 기반 600자 ✅ |
| 4단계 (신규) | kiwi 명사 추출 → keywords_text ✅ |
| 검색 fallback | linked 없으면 keyword 검색 ✅ |
| 검색 키워드 | 구문 매칭 + AND 조건 강화 ✅ |
| tsvector | plainto_tsquery AND → to_tsquery OR + kiwi ✅ |
| JSON 추출 | fast_extract → Haiku LLM ✅ |
| 로그 | 파이프라인 단계별 로깅 (로컬) ✅ |
